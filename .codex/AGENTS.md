# Codex Repository Rules

## Git Operations

For this repository, `git add` and `git commit` are approved as routine, non-destructive actions when they are needed to complete the user's requested work.

- Do not ask for separate confirmation before running `git add` or `git commit`.
- Keep commits focused and use the repository's Conventional Commits style with Japanese descriptions.
- Do not run destructive Git commands such as `git reset --hard`, `git checkout --`, or history rewriting commands unless the user explicitly requests that exact action.
- Inspect the working tree before committing and include only files related to the current task.
